Chapter 2

Variables

Though var is used in old examples of code, using const and let are the new standard. We should try and use const as much as possible as this will help to avoid problems down the road.

const and let both have 'block scope' so they only exist inside blocks they are declared in."variables defined inside a block using the let or const will only be available inside that block"

Using global scope is not considered to be good practice.

Naming

When naming constants and Variables you should be as sensible as you can, naming a variable something 'tablet', 'mouse', or 'bottle' is better than naming something 'x', or 'y' or '123'


Examples of Valid names

  • $name
  • _answer
  • firstName
  • last_name
  • address_line1

Chapter 3

To create Arrays you can use either array literals "ex: const myArray = [];" or the array constructor function "ex: const myArrya = new Array();". It is preferable to use array literls over array constructor to use less typing.

Maps are a type of list with key value pairs, similar to 'dictionaries' in other programming languages. Maps can use any data type as a key. At this time there is no literal way to creat a map, "ex: const romanNumerals = new Map();"

This symbol ( => ) is known as a 'hash rocket'.

Chapter 4

There are several different ways to declare a function, it is not recommended to declare functions using the constructor method as this can cause unexpected errors.

  • Function() Declaration

    function hello(){ console.log('Hello World!'); }

  • Function() Expression

    const goodbye = function(){ console.log('Goodbye World!'); };

  • Function() Constructor

    const hi = new Function('console.log("Hi World!");');

In ES6 there is a new way to declare a function, it is called the arrow syntax, the arrow symbol is also know as a 'hash rocket'
"ex: const square = x => x*x;"

Advantages of Arrow Functions

  • They are much less verbose than normal function declarations.
  • Single parameters don't need putting into parentheses.
  • The body of the function doesn't need placing inside a block if it's only one line.
  • The return keyword isn't required if the return statement is the only statement in the body of the function.
  • They don't bind their own value of this to the function (we'll see why this is a particularly useful property when we cover objects later in the book).
